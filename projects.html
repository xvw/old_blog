Retrouvez ici la liste des projects plus ou moins maintenus. Même si peu d'entre eux sont terminés, je tâcherai de n'en abandonner aucun en cours de route ! Projets viables

Cette liste propose des projets déjà utilisables (bien que pour la plupart, non terminé). N'hésitez pas à contribuer sur l'envie vous prend !

Site webAtelier-prog

Atelier-prog est un petit site proposant des exercices ordonnés. Il a été réalisé pour, principalement, des débutants, mais tout le monde à le droit de contribuer en ajoutant des exercices ou des propositions de solutions.

  • Html
  • Css
  • +participations OCaml, Erlang, C et bien d'autres
Jeux vidéosRMEx

RMEx est une surcouche au logiciel RPGMaker VXAce. Il offre une suite d'outils pour étendre fortement les possibilités de l'outil. Il offre aussi des outils pour structurer correctement des projets avec RPGMaker.

  • DSL d'extension
  • Une documentation interactive
  • Un gestionnaire de paquets
  • Ruby
  • Win32API
  • JavaScript
  • Erlang
  • Html
  • Css
CadricielPhun

Phun est un cadriciel rédigé en PHP pour déployer facilement des applications web sur une architecture LAMP. Il propose une architecture orientée service, décrite par des URL's. Actuellement, Phun est avant tout un système de routing expressif.

  • PHP
BibliothèqueOCaml Batteries

Batteries est une bibliothèque standard pour OCaml (réalisé par la communauté OCaml). Mes contributions sur le sujet auront principalement portées sur l'implémentation du module Substring, notamment dans l'enrichissement des fonctionnalités et l'implémentation de tests unitaires.

  • OCaml
BibliothèqueCoers

Coers est une toute petite bibliothèque de conversion de données pour le langage Erlang. Elle sert beaucoup de projets internes à l'entreprise qui m'emploie.

  • Erlang
BibliothèqueJeSON

JeSON permet de sérialiser/déserialiser des records Erlang vers ou via du JSon. Elle repose sur le principe de générer une fonction d'encodage ou de décodage de JSon. Elle utilise une interface typée.

  • Erlang

Expérimentations

Cette énumération propose les projets "jouets", qui ne servent plus énormément ou qui ne sont pas utilisables.

Bibliothèque Drumaderian

Une bibliothèque en cours de développement pour permettre la création de jeux-vidéos en WebGL (mais tout de même orienté 2D) dans le navigateur. L'ensemble de la bibliothèque est rédigée en OCaml et compilée vers JavaScript avec Js_of_ocaml.
Ce projet s'inspire de beaucoup de travail effectué dans Js_of_ocaml_bootstrapper.

  • OCaml
  • Js_of_OCaml
  • WebGL
Outil Erl-test-runner

Ce projet rend l'utilisation de Travis avec Erlang pour des modules atomiques très simples et n'impose pas Rebar (ou le comportement OTP application). L'outil est très convaincant pour tester en intégration continue, des modules atomiques.
Depuis, nous avons pris l'habitude d'utiliser Rebar.

  • Erlang
Outil Ppx_test

Ppx_test est un petit préprocesseur qui permet de placer "en inline" des tests unitaires dans un module OCaml. En plus de substituer des portions de code, l'outil propose un tout petit framework de tests unitaires.
Un article résumant son fonctionnement

  • OCaml
Bibliothèque Js_of_ocaml_bootstrapper

Js_of_ocaml_bootstrapper est une bibliothèque pour démarrer facilement un projet JavaScript avec Js_of_ocaml. L'enjeu de cette bibliothèque est de facilement pouvoir prototyper des applications en très peu de temps. Actuellement, l'API propose des sélécteurs, des processus, des promesses et une API riche pour Canvas.

  • OCaml
  • JavaScript
Bibliothèque OML: Ocaml Mini Library

Oml est une toute petite bibliothèque OCaml qui implémente une interface monadique riche pour certaines structures de données. Bien que beaucoup moins riche que Batteries ou Core, je continue ce projet pour me former au design de bibliothèques avec OCaml.

  • OCaml
Bibliothèque Piplet

Piplet est un projet qui avance drastiquement lentement pour permettre la création de pages statiques reposant sur Github. La bibliothèque exporte une série de commandes Unix ainsi qu'un Binding de Pandoc pour la conversion de textes de Markdown à Html ou PDF.

  • OCaml

Jeux-vidéos

Il m'arrive parfois d'avoir envie d'essayer de créer des jeux-vidéos. En voici une liste de ceux que j'accepte de montrer.

Jeux vidéos Almost Heroïc

Un jeu réalisé en 72 heures à l'occasion du Ludum Dare 33, dont le thème était : "You are the monster". Le jeu a été réalisé de toute pièce en 72 heures avec Joke, Zangther, Neha et moi même. Nous nous sommes servi de RPGMaker VXAce et RME. Le jeu est arrivé 60ème (sur plus de 1500 participations).

  • Fiche Ludum Dare
  • Ruby
  • RME
